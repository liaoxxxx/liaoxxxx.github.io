  • 事务是一个最小的工作单元，不论成功与否都作为一个整体进行工作。

    不会有部分完成的事务。由于事务是由几个任务组成的，因此如果一个事务作为一个整体是成功的， 则事务中的每个任务都必须成功。如果事务中有一部分失败，则整个事务失败。

    当事务失败时，系统返回到事务开始时的状态。这个取消所有变化的过程称为“回滚”( rollback )。 例如，如果一个事务成功更新了两个表，在更新第三个表时失败，则系统将两次更新恢复原状， 并返回到原始的状态。

  • 使用场景.

    如商品订单生成操作时加锁,隔离其他的事物操作,避免脏读 ,保证数据的 一致性;

    如保证某个字段的单条记录的唯一性,如表A的操作成功,而接下来的表B的操作因为要保证某个字段的单条记 录的唯一性,表B的操作失败了,不幸的是之前表A的数据已接插入无用的数据,这时就可以使用回滚操作

  • 事务在Thinkphp5.1中的使用。

    使用事务处理的话，需要数据库引擎支持事务处理。比如 MySQL 的 MyISAM 不支持事务处理，需要使用 InnoDB 引擎。

    // 启动事务
Db::startTrans();
try {
    Db::table('think_user')->find(1);
    Db::table('think_user')->delete(1);
    // 提交事务
    Db::commit();
} catch (\Exception $e) {
    // 回滚事务
    Db::rollback();
}

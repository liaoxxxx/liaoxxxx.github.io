  • 如何避免字段重名无法准确获取

    1.在联合查询（join）时，因为设计数据表时不能完全避免两个相关联的表存在相同的字段名，譬'id’

    所以在对这两个表进行联合查询，获取的结果中将无法完全正常获取这两个字段的值，解决方案如下:

     class Manager extends Controller
{
   public function index()
   {
   	    $managers= Db::table('shop_manager')
	        ->alias('m')
                ->field(' m.id as m_id ,nickname,passwd,freeze,phone,email,sex,age')
	        ->join(['shop_manager_detail'=>'m_d'],'m.id=m_d.id')
	        ->join(['shop_manager_role'=>'m_r'],'m.id=m_r.shop_manager_id')
   	        ->select();
   	    $this->assign('manager',$managers);
        return view();
   }

    其中 ->alias('m') 是对上文中的shop_manager 起了别名，所以 m.id 即是 表：shop_manager的id字段; m.id as m_id 就是对该字段 id 更名为 m_id,有效避免了字段名重复
